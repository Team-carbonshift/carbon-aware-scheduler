# jobs.csv — 열 정의 & 통계 (간결판, 8열)

생성기: `gen_jobs.py` (SEED=42, 재현 가능). 시간 값은 전부 **float 초**, UTC 절대축 (t=0 = 2026-01-01 00:00 UTC).
⚠️ 시간 비교는 `==` 금지, 허용오차 사용 (CSV 재독 시 float 노이즈).

## 열 정의

| 열 | 단위 | 설명 |
|---|---|---|
| `job_name` | — | 식별자 |
| `submit_time` | 초 (UTC) | 제출 시각 |
| `duration` | 초 | 실행 길이. k별 로그정규 + 클램프 |
| `region` | — | LB 배정 리전 (Azure 8개, LB 표기) |
| `k` | — | 중요도 5(급함)~1(여유). 전역 비율 3:2:1:1:3 |
| `L_max` | 초 | 지연 예산. **k별 범위 내 job마다 log-uniform 랜덤** |
| `submit_local_hour` | 시 (0~24, 현지) | 검증용. 낮밤 7:3 확인 |
| `band` | — | 검증용. 생성 시 시간대 (day = 현지 8~20시) |

## 유도 규칙 (저장 안 함 — 스케줄러 코드에서 계산)

```
t_earliest = submit_time
t_latest   = submit_time + L_max
deadline   = submit_time + L_max + duration     (규약: L_max = 지연예산)
alpha      = {5:0.2, 4:0.4, 3:0.6, 2:0.8, 1:1.0}[k]
deferrable = (L_max >= 3600)
```

## k 정책표

| k | 작업 예시 | L_max 범위 | duration 중앙 (σ) | dur 클램프 | 비율 |
|---|---|---|---|---|---|
| 5 | 웹페이지 로딩, API 응답, 결제 | 1~5초 | 5초 (0.6) | 1초~1분 | 30% |
| 4 | 검색 결과, 업로드 확인 | 15~60초 | 30초 (0.7) | 1초~5분 | 20% |
| 3 | 주문상태 업데이트, 배송알림 | 2~10분 | 3분 (0.8) | 5초~30분 | 10% |
| 2 | 정산배치, DB백업, 재고동기화 | 2~8시간 | 30분 (0.9) | 1분~4시간 | 10% |
| 1 | 로그정리, 통계리포트, 모델재학습 | 12~24시간 | 1.5시간 (0.9) | 5분~6시간 | 30% |

L_max 랜덤을 고정값으로 되돌리려면 생성기 `L_MAX_RANDOM = False`.

## 통계 (7일분, 2,800개)

```
k 분포        : 31.0 / 19.2 / 10.3 / 10.3 / 29.2 %   (목표 30/20/10/10/30)
L_max 실측    : 각 k 범위 안에 정확히 들어감 (assert 통과)
낮밤 7:3      : 주간형 69.1% / 야간형 69.0%           (목표 70%)
deferrable    : 1,105개 (39.5%)  ← L_max ≥ 1h, 시간이동 대상
리전          : 8리전 × 350개 균등
```

## 샘플링 규칙 요약

k-first (k 먼저 3:2:1:1:3) → 제출시각: 주간형(k=5,4,3)은 현지 낮 70%(13시 봉우리 σ=3h)+밤 30%(균등),
야간형(k=1,2)은 밤 70%(02시 봉우리)+낮 30%(균등). 낮 = 현지 [8, 20)시. 하드 컷 없음.
